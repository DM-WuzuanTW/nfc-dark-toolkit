# 🚀 快速開始指南

## 📋 前置要求

### 硬體需求
- ✅ 支援 NFC 的 Android 裝置（實體機）
- ✅ USB 連接線

### 軟體需求
- ✅ Android Studio Koala (2023.3.1) 或更高版本
- ✅ JDK 17
- ✅ Android SDK (API 26-34)

## 🔧 安裝步驟

### 1. 克隆專案

```bash
git clone https://github.com/WuzuanTW/nfc-dark-toolkit.git
cd nfc-dark-toolkit
```

### 2. 用 Android Studio 開啟

1. 開啟 Android Studio
2. 選擇 `File > Open`
3. 選擇專案目錄
4. 等待 Gradle 同步完成

### 3. 同步 Gradle

如果自動同步失敗：
- 點擊 `File > Sync Project with Gradle Files`

### 4. 連接裝置

1. 在 Android 裝置上開啟「開發者選項」
2. 啟用「USB 偵錯」
3. 用 USB 線連接到電腦
4. 授權 USB 偵錯

### 5. 執行應用程式

1. 在 Android Studio 中點擊 `Run > Run 'app'`
2. 選擇您的裝置
3. 等待應用程式安裝並啟動

## 📱 首次使用

### 1. 檢查 NFC 狀態

應用程式啟動後會自動檢查：
- 裝置是否支援 NFC
- NFC 是否已啟用

如果 NFC 未開啟，會顯示提示並引導您到設定頁面。

### 2. 掃描第一個 NFC 標籤

1. 確保 NFC 已啟用
2. 點擊底部「掃描」圖示
3. 將手機背面靠近 NFC 標籤
4. 等待資料讀取完成
5. 查看標籤詳細資訊

### 3. 查看標籤資訊

掃描成功後，您會看到：
- **標籤 ID**: 唯一識別碼
- **標籤類型**: 如 MIFARE_ULTRALIGHT, NTAG 等
- **技術列表**: 支援的 NFC 技術
- **可寫入狀態**: 標籤是否可寫入
- **容量資訊**: 已使用/總容量
- **NDEF 內容**: 標籤內儲存的資料

## 🧪 測試功能

### 測試掃描功能

**需要準備**：
- NFC 標籤（建議 NTAG213 或 Mifare Ultralight）

**測試步驟**：
1. 開啟應用程式
2. 進入「掃描」頁面
3. 靠近標籤
4. 確認能正確顯示標籤資訊

**預期結果**：
- ✅ 顯示標籤 ID（格式如：04:E2:3C:AA:5B:28:80）
- ✅ 顯示標籤類型
- ✅ 顯示技術列表
- ✅ 顯示可寫入狀態
- ✅ 如果有 NDEF 資料，顯示內容

### 測試歷史記錄

**測試步驟**：
1. 掃描幾個不同的標籤
2. 點擊底部「歷史」圖示
3. 查看記錄是否儲存（目前顯示「開發中」）

## ⚠️ 常見問題

### 1. NFC 不工作

**問題**: 應用程式無法偵測 NFC 標籤

**解決方案**:
- 確認裝置支援 NFC
- 確認 NFC 已在系統設定中啟用
- 確認標籤放置位置（通常在手機背面中央）
- 嘗試慢慢移動標籤找到最佳位置

### 2. Gradle 同步失敗

**問題**: 無法同步 Gradle

**解決方案**:
```bash
# 清除快取並重建
./gradlew clean
./gradlew build --refresh-dependencies
```

### 3. 應用程式閃退

**問題**: 開啟應用程式後立即閃退

**解決方案**:
- 檢查 Logcat 錯誤訊息
- 確認裝置系統版本 >= Android 8.0
- 重新安裝應用程式

### 4. 標籤讀取失敗

**問題**: 靠近標籤但無反應

**解決方案**:
- 確認標籤沒有損壞
- 嘗試其他標籤
- 確認標籤類型受支援
- 檢查是否有金屬物體干擾

## 💡 小技巧

### 快速測試
1. 使用空白的 NTAG213 標籤進行測試
2. 可以從淘寶/蝦皮購買便宜的 NFC 貼紙
3. 某些門禁卡也可以用來測試（不要寫入！）

### 開發建議
1. 使用實體機測試，模擬器不支援 NFC
2. 保持標籤靠近手機 2-3 秒
3. 注意手機殼可能會影響訊號

### 安全提醒
⚠️ **不要用本應用程式複製以下卡片**：
- 門禁卡（除非您擁有）
- 信用卡/金融卡
- 交通卡
- 其他受保護的智慧卡
